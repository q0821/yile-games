# PRD — 象棋對弈體驗優化

對象：`#xiangqi` 模式（`xiangqi-{ui,mode}.js` + index.html/style.css）。
目標：把「能下」提升到「好下、好看、看得懂」。不動棋規（ffish）與引擎（Fairy-Stockfish）。

## 需求與驗收標準

### R1 棋盤與棋子美化（立體實體感、不被切）
- **問題**：棋盤偏擠；棋子是扁平圓形、無厚度；邊角棋子貼著／被棋盤邊界切到。
- **要做**：
  - 加大內距 padding，確保**最邊緣的棋子完整落在棋盤內、不被邊框切到**（padding ≥ 棋子半徑 + 安全邊距）。
  - 棋子改為**實體立體感**：徑向漸層（左上亮、右下暗）做出圓潤凸面、外圈深色描邊 + 內圈、左上高光，棋盤上落下**柔和陰影**。
  - 紅黑配色維持（紅子紅字、黑子墨字），質感像實體木棋子。
- **驗收**：截圖檢視四角棋子皆完整不被切；棋子有可見陰影與凸面漸層；整體不顯擠。

### R2 AI 思考延遲 + 讀取效果
- **問題**：AI 落子太快，沒有思考感。
- **要做**：
  - AI 回手前總時間 **1–3 秒**（含引擎實際運算；不足的差額補等待）。
  - 顯示**思考中讀取效果**：狀態列「電腦思考中…」+ 動畫指示（旋轉／脈動 spinner）。
- **驗收**：人走子後，AI 約 1–3 秒才落子；期間有明顯動畫讀取效果。

### R3 棋子移動動畫
- **問題**：棋子瞬間跳位，看不出從哪移到哪。
- **要做**：落子（人 + AI）時，棋子從起點**滑動**到終點（約 220–320ms ease）。吃子時被吃棋子於滑動結束時消失。
- **驗收**：每次走子看得到棋子平滑移動；位置正確；動畫期間不可再操作（避免競態）。

### R4 將軍與結束明顯提示
- **問題**：將軍無明顯提示易忽略；對局結束不明顯。
- **要做**：
  - **將軍**：被將一方的「將／帥」以**紅色脈動高亮**標示，並短暫顯示「將軍！」提示。
  - **結束**：以**覆蓋整個棋盤的結束畫面**呈現「紅方勝／黑方勝／和局」+「再來一局」按鈕；點按重新開始。
- **驗收**：將軍時將帥明顯閃示 + 文字提示；分出勝負時跳出結束畫面，按鈕可重新開局。

## 範圍外（本次不做）
- 悔棋、存檔、難度微調、NNUE、棋譜記錄。

## 實作規劃（階段）
1. **R1 美化**：xiangqi-ui.js 棋子 3D 渲染 + padding；視覺驗收。
2. **R2 思考延遲/讀取**：xiangqi-mode.js 最小延遲 + spinner（CSS）。
3. **R3 移動動畫**：drawXiangqi 支援「隱藏某格 + 浮動棋子」；mode 用 rAF 跑 from→to 插值，動畫中鎖操作。
4. **R4 將軍/結束**：被將將帥高亮 + 「將軍！」提示；結束覆蓋畫面（index.html + CSS + mode 觸發）。

每階段 dev 實機截圖驗收；全部完成跑既有測試後 commit。
